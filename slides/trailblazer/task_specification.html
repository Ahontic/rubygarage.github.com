Trailblazer Workshop example/task specification

Example description:

API application with next set of user management features:

  • Administrator views the list of users
  • Administrator deletes a user
  • User restores his password
  • Send a restore password email
  • Verify a token
  • Restore a password by token
  • User changes his password

Implemented with following assumptions:

  • User is global to the application;
  • Role management mocked with is_admin field on User model;
  • User sign up is out of scope of this example, so users is preseeded
  • User authentication/session mocked with “Authentication: user_id: {id}” header

Task description:

Extend example with following features:

  • Administrator invites a user
  • User signs up from invitation
  • User logs in
  • User logs out

Requirements:

  • User authentication shall be implemented via JWT token using ruby-jwt gem, or [add alternatives].

Example specifications

User story 1 - View the list of Users

As an Administrator

I want to be able to see the list of Users

So that I could see who is using my application

Scenario 1.1: Administrator views the list of Users

Given I’m authorized as Administrator

When I navigate to the Users page

Then I see the Users List

And the Users List consists of all users sorted by last name in ascending order

And I see 20 users per page

User story 2 - Delete a user

As an Administrator

I want to be able to delete a User

So that user will no longer have access to my application

Scenario 2.1: Administrator deletes a user

Given I’m authorized as Administrator

And some users are already registered in the application

And I’m on the Users page

When I click on the Delete button next to some User

Then the system deletes User’s account from the database

And the system sends an email notification Your account has been deleted to deleted User

Scenario 2.2: Administrator deletes self

Given I’m authorized as Administrator

And some users are already registered in the application

And I’m on the Users page

When I click on the Delete button next to myself

Then the system doesn’t delete my account

And I see the You can’t delete yourself error

User story 3 - Restore a password

As a User

I want to be able to restore forgotten password

So that I can access my account in case if I forgot my current password

Scenario 3.1: User requests a password restore

Given I’m in a role of a Guest User

And I have a User account in the system

And I’m in the Login page

When I click the “Forgot your password?” button

Then I’m navigated to the Forgot Password page

When I fill in Forgot password form 

And I click the Reset Password button

Then I see a confirmation that the reset password link has been sent to my email

And I receive the Email notification with the restore password instructions

Technical Details: back-end should have separate endpoint to validate token before front-end shows reset form to prevent user from excessive data input
